## 2. מצב הקבצים בתיקייה
```
/
├── index.html      # הדף המלא — 13 סקשנים, סמנטי, RTL, נגיש. מוכן.
├── styles.css      # עיצוב המותג, רספונסיביות (mobile-first), אנימציות. מוכן.
├── script.js       # reveal, count-up, אקורדיון, קרוסלה, ולידציה ושליחת טופס. מוכן.
├── render.yaml     # קונפיג פריסה ב-Render כאתר סטטי. מוכן.
├── README.md       # מדריך פריסה מלא למשתמש (GitHub → Render → טופס → פיקסל).
├── COWORK.md       # הקובץ הזה.
└── assets/
    └── logo.png    # הלוגו האמיתי של החברה (שקוף). משובץ בהדר ובפוטר.
```

**אין שלב build** — זה Vanilla HTML/CSS/JS. פותחים את `index.html` בדפדפן והכל עובד.
הרצה מקומית לבדיקה: `python -m http.server 8000` ואז `http://localhost:8000`.

## 6. מה נשאר לעשות (סדר עדיפויות)

### א. העלאה ל-GitHub
1. אתחל git בתיקייה, צור commit עם כל הקבצים.
2. צור repo חדש (ציבורי) בחשבון ה-GitHub של המשתמש ודחוף אליו את הקוד.
   (שאל את המשתמש לשם המשתמש ב-GitHub ולשם הרצוי ל-repo אם לא ידוע.)

### ב. פריסה ב-Render כאתר סטטי
- יש כבר `render.yaml` מוכן (Static Site, publish path = `.`).
- הנחה את המשתמש: render.com → New → Static Site (או Blueprint) → לבחור את ה-repo →
  Build Command ריק, Publish Directory = `.` → Create.
- התוצאה: כתובת חיה `https://<name>.onrender.com`.

### ג. חיבור שליחת הטופס (חשוב!)
- ב-`index.html`, בתוך `<form id="leadForm" action="...">`, יש placeholder: `YOUR_FORM_ID`.
- ברירת המחדל המתועדת: **Formspree** — להחליף `https://formspree.io/f/YOUR_FORM_ID`
  במזהה אמיתי, ולהגדיר שהלידים יישלחו ל-`admateinu.beitenu@gmail.com`.
- שמות השדות בטופס כבר בעברית ותואמים לפורמט הלידים (שם, טלפון, עיר, סוג הצורך,
  בעל נכס, סכום מבוקש, LeadID) — שמור על המבנה הזה גם אם מחברים ל-CRM/webhook אחר.
- עד שמוגדר endpoint, הטופס מציג הודעת הצלחה ופותח אוטומטית וואטסאפ עם הפרטים (fallback) —
  אל תשבור את ה-fallback הזה.

### ד. פיקסל פייסבוק (הקמפיינים רצים בפייסבוק)

## 7. דרישות איכות שחובה לשמור עליהן בכל עריכה
- **RTL** מלא, **עברית תקנית** בלבד בכל טקסט שמופיע בדף.
- **Mobile-first ורספונסיבי** — לבדוק מובייל (<768), טאבלט (768–1024), דסקטופ (>1024).
  קהל היעד גולש בעיקר במובייל (פייסבוק).
- **נגישות (a11y)**: סמנטי, alt לתמונות, aria-labels לכפתורים הצפים ולאקורדיון, ניגודיות, ניווט מקלדת.
- שלושת ערוצי ההמרה (טופס/חיוג/וואטסאפ) חייבים להישאר זמינים לאורך הדף + הכפתורים הצפים + רצועת המובייל.
- קוד נקי עם הערות. בלי פריימוורקים כבדים (להשאיר Vanilla).
