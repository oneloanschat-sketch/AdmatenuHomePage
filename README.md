# אדמתנו ביתנו — דף נחיתה (משכנתאות ואיחוד הלוואות)

דף נחיתה ממיר, בעברית מלאה (RTL), מוכן לפריסה כ‑**Static Site** ב‑Render.
בנוי ב‑Vanilla HTML/CSS/JS בלבד — בלי פריימוורקים, בלי שלב build.

3 ערוצי המרה: **טופס לידים · חיוג 054‑555‑4588 · צ'אט בוט וואטסאפ 058‑755‑4588**.

## 🚀 העלאה לאוויר — מדריך צעד אחר צעד (למתחילים)

### שלב 1 — פתיחת חשבון GitHub והעלאת הקבצים

1. היכנסו ל‑[github.com](https://github.com) והירשמו (חינם).
2. לחצו על הכפתור הירוק **New** ליצירת מאגר (Repository) חדש.
3. תנו שם, למשל `admateinu-landing`, השאירו **Public**, ולחצו **Create repository**.
4. במסך הבא לחצו על הקישור **uploading an existing file**.
5. **גררו את כל הקבצים** של הפרויקט (כולל תיקיית `assets`) לתוך החלון.
6. למטה לחצו **Commit changes**. זהו — הקוד שלכם ב‑GitHub. ✅

> אם נוח לכם בשורת פקודה, אפשר במקום זה:
> ```bash
> git init
> git add .
> git commit -m "אדמתנו ביתנו - דף נחיתה"
> git branch -M main
> git remote add origin https://github.com/USERNAME/admateinu-landing.git
> git push -u origin main
> ```

### שלב 2 — חיבור ל‑Render כ‑Static Site

1. היכנסו ל‑[render.com](https://render.com) והירשמו עם **חשבון GitHub** (Sign up with GitHub).
2. בלוח הבקרה לחצו **New +** ← **Static Site**.
3. בחרו את המאגר `admateinu-landing` (אשרו ל‑Render גישה אם תתבקשו).
4. מלאו את ההגדרות:
   - **Name:** `admateinu-beitenu` (או כל שם)
   - **Branch:** `main`
   - **Build Command:** השאירו **ריק**
   - **Publish Directory:** `.` (נקודה אחת = שורש הפרויקט)
5. לחצו **Create Static Site**.
6. המתינו דקה — Render בונה ומפרסם. תקבלו כתובת חיה כמו:
   `https://admateinu-beitenu.onrender.com` 🎉

> מאחר שיש בפרויקט קובץ `render.yaml`, אפשר גם לבחור **New + ← Blueprint** ו‑Render יקרא ממנו את כל ההגדרות אוטומטית.

**עדכון האתר בעתיד:** כל שינוי שתעלו ל‑GitHub (Commit) — Render מפרסם אותו אוטומטית.

### שלב 3 — דומיין משלכם (אופציונלי)

ב‑Render: בעמוד האתר ← לשונית **Settings** ← **Custom Domains** ← הוסיפו את הדומיין שלכם ועקבו אחרי ההוראות לעדכון רשומות ה‑DNS אצל ספק הדומיין.

## 📨 חיבור שליחת הטופס (חשוב!)

הטופס כבר מחובר ל‑**Web3Forms** (שירות חינמי שמתאים לאתר סטטי). נשאר רק להדביק מפתח אישי אחד.
עד שלא תגדירו את המפתח — הטופס יראה הודעת הצלחה ויפתח אוטומטית וואטסאפ עם הפרטים (fallback), כך שלא מאבדים אף פנייה.

### אפשרות א' (ברירת המחדל, מומלצת) — Web3Forms

1. היכנסו ל‑[web3forms.com](https://web3forms.com), הזינו את המייל **admateinu.beitenu@gmail.com** ולחצו ליצירת מפתח. תקבלו במייל **Access Key** (מחרוזת ארוכה).
2. ב‑`index.html` מצאו את השורה:
   ```html
   <input type="hidden" name="access_key" value="YOUR_WEB3FORMS_ACCESS_KEY" />
   ```
   והחליפו את `YOUR_WEB3FORMS_ACCESS_KEY` במפתח שקיבלתם (השאירו את הגרשיים).
3. שמרו, העלו ל‑GitHub — וזהו. כל ליד יגיע למייל בפורמט מובנה:
   **שם · טלפון · עיר · סוג הצורך · בעל נכס · סכום מבוקש · LeadID**.

> אין צורך לשנות את ה‑`action` — הוא כבר מוגדר ל‑`https://api.web3forms.com/submit`.
> שדה ה‑`botcheck` המוסתר בטופס הוא הגנת ספאם של Web3Forms — אל תסירו אותו.

### אפשרות ב' — Formspree (חלופה)

1. היכנסו ל‑[formspree.io](https://formspree.io), הירשמו וצרו טופס — תקבלו מזהה כמו `xayzwabc`.
2. ב‑`index.html` שנו את ה‑`action` ל‑`https://formspree.io/f/xayzwabc`, והחליפו ב‑`script.js` את הבדיקה כך שתזהה את הכתובת כ‑configured (או פשוט השאירו את שדה ה‑`access_key` ריק/מחקו אותו).

### אפשרות ג' — חיבור עתידי ל‑CRM / webhook

החליפו את ה‑`action` בכתובת ה‑webhook של מערכת ה‑CRM שלכם.
שמות השדות בעברית כבר תואמים לפורמט הלידים הקיים (שם, טלפון, עיר, מקור/סוג הצורך, בעל נכס, סכום מבוקש, LeadID), כך שקל למפות אותם.

## 📊 פיקסל פייסבוק ומדידת המרות (לקמפיינים)

הקמפיינים רצים בפייסבוק — מומלץ מאוד להתקין פיקסל למדידת לידים.

1. ב‑`index.html` מצאו את הבלוק המסומן **META PIXEL** (בתוך `<head>`).
2. הסירו את סימני ההערה (`<!--` ו‑`-->`) שמסביב לקוד.
3. החליפו את `YOUR_PIXEL_ID` במזהה הפיקסל שלכם.

האירועים נשלחים אוטומטית: `PageView` בטעינה, ו‑**`Lead`** בשליחת הטופס.
כן נשלחים אירועים מותאמים על כל קליק חיוג/וואטסאפ/CTA (דרך `data-track`).

> רוצים Google Tag Manager במקום? יש בלוק **GOOGLE TAG MANAGER** מוכן באותו אזור.
